Adaptive Images for Dark Mode in DocFX

Overview

This document describes the CSS-based approach for dynamically handling dark images in light/dark themes without maintaining separate image versions.

Implementation

CSS Rule Added to main.css

Universal White Background for Icons

/* Add white background to all integration icons in dark mode */
[data-bs-theme="dark"] img[style*="height:20px"] {
  background-color: white;
  padding: 2px;
  border-radius: 4px;
}

This rule automatically adds a white background to all icons that have style="height:20px" (which includes all integration icons) when in dark mode.

Usage

All images with style="height:20px" automatically get a white background in dark mode. This includes:

  • All integration icons in the tables
  • Any other small icons with the same height styling

No additional classes or markup changes are required.

How It Works

  1. White Background: Adds a white background behind icons in dark mode
  2. Padding: 2px padding ensures the icon doesn't touch the background edge
  3. Border Radius: 4px radius provides subtle rounded corners
  4. Theme Detection: [data-bs-theme="dark"] applies rules only in dark mode

Benefits

  • No duplicate images to maintain
  • Automatic handling for common patterns
  • Flexible manual control when needed
  • Pure CSS solution (no JavaScript required)
  • Works with DocFX's built-in theme switching

Testing

To test the implementation:

  1. Build the DocFX site: docfx build --serve
  2. Navigate to /docs/resources/integrations.html
  3. Toggle between light and dark themes
  4. Verify dark logos are visible in both modes

Adding New Patterns

To add automatic inversion for new image patterns, edit /public-documentation/xmpro-template/public/main.css:

[data-bs-theme="dark"] img[src*="new-dark-logo"] {
  filter: invert(0.85) hue-rotate(180deg);
}

Notes

  • The invert(0.85) value can be adjusted (0-1) for different inversion strengths
  • Some logos with specific brand colors should be excluded from inversion
  • This approach works best with monochrome or simple logos
  • Complex images with gradients may need custom handling
