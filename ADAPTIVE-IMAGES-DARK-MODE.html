Adaptive Images for Dark Mode in DocFX

Overview

This document describes the CSS-based approach for dynamically handling dark images in light/dark themes without maintaining separate image versions.

Implementation

CSS Rules Added to main.css

  1. Manual Classes for Flexible Control

    /* Automatically invert dark logos/icons in dark mode */
[data-bs-theme="dark"] .auto-invert {
  filter: invert(1) hue-rotate(180deg);
}

/* For images that need brightness adjustment */
[data-bs-theme="dark"] .auto-brighten {
  filter: brightness(0) invert(1);
}

  2. Automatic Rules for Common Dark Logos

    /* Specific adjustments for integration logos */
[data-bs-theme="dark"] img[src*="openai-logo"],
[data-bs-theme="dark"] img[src*="Azure AI"],
[data-bs-theme="dark"] img[src*="AzureSQL"],
[data-bs-theme="dark"] img[src*="Azure ML"],
[data-bs-theme="dark"] img[src*="azure"],
[data-bs-theme="dark"] img[src*="mongodb"],
[data-bs-theme="dark"] img[src*="mysql"] {
  filter: invert(0.85) hue-rotate(180deg);
}

  3. Preserve Colorful Logos

    /* Preserve colorful logos that should not be inverted */
[data-bs-theme="dark"] img[src*="Salesforce"],
[data-bs-theme="dark"] img[src*="Twilio"],
[data-bs-theme="dark"] img[src*="ifm"],
[data-bs-theme="dark"] img[src*="MOVUS"],
[data-bs-theme="dark"] img[src*="Amber"] {
  filter: none;
}

Usage

Option 1: Automatic (Already Applied)

Images with filenames containing "azure", "openai-logo", "mongodb", etc. will automatically be inverted in dark mode.

Option 2: Manual Classes

For new dark images that need inversion:

<img src="images/dark-logo.png" class="auto-invert" style="height:20px;">

For images needing brightness adjustment:

<img src="images/dark-icon.png" class="auto-brighten" style="height:20px;">

How It Works

  1. CSS Filters: Uses filter: invert() to flip dark pixels to light
  2. Hue Rotation: hue-rotate(180deg) preserves color relationships after inversion
  3. Attribute Selectors: img[src*="pattern"] targets images by filename patterns
  4. Theme Detection: [data-bs-theme="dark"] applies rules only in dark mode

Benefits

  • No duplicate images to maintain
  • Automatic handling for common patterns
  • Flexible manual control when needed
  • Pure CSS solution (no JavaScript required)
  • Works with DocFX's built-in theme switching

Testing

To test the implementation:

  1. Build the DocFX site: docfx build --serve
  2. Navigate to /docs/resources/integrations.html
  3. Toggle between light and dark themes
  4. Verify dark logos are visible in both modes

Adding New Patterns

To add automatic inversion for new image patterns, edit /public-documentation/xmpro-template/public/main.css:

[data-bs-theme="dark"] img[src*="new-dark-logo"] {
  filter: invert(0.85) hue-rotate(180deg);
}

Notes

  • The invert(0.85) value can be adjusted (0-1) for different inversion strengths
  • Some logos with specific brand colors should be excluded from inversion
  • This approach works best with monochrome or simple logos
  • Complex images with gradients may need custom handling
