Detailed Analysis

1. Invalid File Links (209 warnings)

Link Type Breakdown:

  • Directory links: 85 (40.7%)

    • Links ending with / that point to directories
    • Example: ~/docs/blocks-toolbox/layout/

  • Markdown files: 41 (19.6%)

    • Links to .md files that don't exist
    • Example: ~/docs/how-tos/publish/admin-unpublish-override.md

  • Image files: 39 (18.7%)

    • Missing .png, .gif, .jpg files
    • Example: ~/docs/images/image-420.png

  • Other/Unknown: 44 (21.0%)

    • Various other file types or malformed links

Common Issues with Invalid File Links:

  1. Using tilde (~) paths instead of relative paths

    • Current: ~/docs/images/image-420.png
    • Should be: ../../images/image-420.png

  2. Missing images

    • Many numbered images are missing (e.g., image-420.png, image-1757.png)
    • Special case: Case-sensitive image extensions (.PNG vs .png)

  3. Links to directories instead of index files

    • Current: ~/docs/blocks-toolbox/layout/
    • Should be: ~/docs/blocks-toolbox/layout/index.md

  4. Non-existent files

    • admin-unpublish-override.md referenced in multiple places but doesn't exist
    • README.md files referenced but should be index.md

2. Invalid Bookmarks (88 warnings)

Files with Most Invalid Bookmarks:

  1. release-notes/archived/v4.1.0.md - 12 warnings
  2. release-notes/archived/v4.1.13.md - 10 warnings
  3. installation/deployment/aws.md - 10 warnings
  4. installation/deployment/windows-server-2022/upgrade.md - 9 warnings

Common Bookmark Issues:

  1. Incorrect anchor formatting

    • Example: #using-variables.xv-file should probably be #using-variablesxv-file
    • Periods and special characters in anchors cause issues

  2. Mismatched heading text

    • Link expects: #properties-on-the-rule
    • Actual heading might be: #rule-properties

  3. Cross-file bookmark references

    • Links trying to reference anchors in other files
    • Format: <a href="path/to/file.md#anchor"> not working properly
