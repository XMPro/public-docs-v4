1. Invalid File Links (209 warnings)

1.1 Tilde Path Issues (~85 occurrences)

Problem: Links use ~/docs/ instead of relative paths Impact: DocFX cannot resolve these paths correctly

Examples:

Current:  ![Image](~/docs/images/image-420.png)
Should be: ![Image](../../images/image-420.png)

Fix Script:

# fix-tilde-paths.sh
#!/bin/bash

# Find all markdown files and fix tilde paths
find docs -name "*.md" -type f | while read file; do
    # Backup original
    cp "$file" "$file.bak"
    
    # Calculate relative path depth
    depth=$(echo "$file" | tr -cd '/' | wc -c)
    depth=$((depth - 1))  # Subtract 1 for 'docs' folder
    
    # Build relative path prefix
    prefix=""
    for ((i=0; i<depth; i++)); do
        prefix="../$prefix"
    done
    
    # Replace tilde paths
    sed -i "s|~/docs/|${prefix}|g" "$file"
done

1.2 Missing Images (39 files)

Problem: Referenced images don't exist in the images directory Categories:

  • Numbered images (e.g., image-420.png, image-1757.png)
  • Case-sensitive extensions (.PNG vs .png)
  • Special named images (e.g., menu-collapse.gif)

Investigation Steps:

  1. Check GitBook assets directory for originals
  2. Map GitBook names to kebab-case names
  3. Copy missing images with correct names

Fix Script:

# find-and-fix-missing-images.sh
#!/bin/bash

GITBOOK_ASSETS="/home/work/projects/xmpro-development-ubuntu/public-documentation/gitbook/docs/assets"
DOCS_IMAGES="/home/work/projects/xmpro-development-ubuntu/public-documentation/docs/images"

# Extract all missing images from warnings
grep "InvalidFileLink.*\(image\|\.png\|\.gif\|\.jpg\)" /tmp/docfx_warnings.txt | \
    grep -o "[^/]*\.\(png\|gif\|jpg\|PNG\)" | sort -u > missing_images.txt

# Process each missing image
while read img; do
    echo "Processing: $img"
    
    # Try different naming patterns in GitBook
    base_name="${img%.*}"
    extension="${img##*.}"
    
    # Search patterns
    find "$GITBOOK_ASSETS" -iname "*${base_name}*" -o -iname "${img}" | while read found; do
        echo "  Found: $found"
        # Copy with correct name
        cp "$found" "$DOCS_IMAGES/$img"
    done
done < missing_images.txt

1.3 Directory Links (85 occurrences)

Problem: Links point to directories instead of index.md files Pattern: Links ending with /

Examples:

Current:  [Layout Blocks](~/docs/blocks-toolbox/layout/)
Should be: [Layout Blocks](~/docs/blocks-toolbox/layout/index.md)

Fix Script:

# fix-directory-links.sh
#!/bin/bash

find docs -name "*.md" -type f | while read file; do
    # Fix directory links - add index.md
    sed -i 's|\(](.*\)/)|]\1/index.md)|g' "$file"
done

2. Invalid Bookmarks (88 warnings)

2.1 Special Characters in Anchors

Problem: Anchors contain periods, parentheses, or other special characters Example: #using-variables.xv-file should be #using-variablesxv-file

Fix Process:

  1. Identify all problematic anchors
  2. Update both the heading and the link

Investigation Script:

# find-invalid-anchors.sh
#!/bin/bash

# Extract all invalid bookmark warnings
grep "InvalidBookmark" /tmp/docfx_warnings.txt > invalid_bookmarks.txt

# Parse and categorize
while IFS= read -r line; do
    file=$(echo "$line" | grep -o "/[^(]*\.md")
    anchor=$(echo "$line" | grep -o "#[^']*")
    echo "$file | $anchor"
done < invalid_bookmarks.txt | sort -u > bookmark_issues.csv

2.2 Mismatched Heading Text

Problem: Link anchors don't match actual heading IDs Common patterns:

  • Plural vs singular
  • Different word order
  • Missing or extra words

Validation Script:

# validate-anchors.sh
#!/bin/bash

# For each file with bookmark warnings
grep "InvalidBookmark" /tmp/docfx_warnings.txt | \
    grep -o "/[^(]*\.md" | sort -u | while read file; do
    
    echo "=== Checking $file ==="
    
    # Extract all headings and generate expected anchors
    grep "^#" "$file" | while read heading; do
        # Convert heading to anchor format
        anchor=$(echo "$heading" | \
            sed 's/^#* //' | \
            tr '[:upper:]' '[:lower:]' | \
            sed 's/[^a-z0-9-]/-/g' | \
            sed 's/-\+/-/g' | \
            sed 's/^-\|-$//g')
        
        echo "  Heading: $heading"
        echo "  Anchor:  #$anchor"
    done
done

2.3 Cross-file References

Problem: Links trying to reference anchors in other files using incorrect syntax

Correct DocFX Syntax:

<!-- Incorrect -->
<a href="path/to/file.md#anchor">Link Text</a>

<!-- Correct -->
[Link Text](path/to/file.md#anchor)

3. Missing YAML Mime (3 warnings)

3.1 Affected Files

  • /public-documentation/azure-pipelines.yml
  • /public-documentation/azure-pipelines-tests.yml
  • /public-documentation/templates/deploy-docs.yml

3.2 Solution

These are CI/CD pipeline files, not documentation. They should be excluded from DocFX processing.

Fix: Update docfx.json

{
  "build": {
    "content": [
      {
        "files": ["**/*.yml", "**/*.md"],
        "exclude": [
          "**/azure-pipelines*.yml",
          "**/templates/**/*.yml",
          "**/obj/**",
          "**/bin/**",
          "_site/**"
        ]
      }
    ]
  }
}

5.2 Continuous Monitoring

# .github/workflows/docfx-validation.yml
name: DocFX Validation

on:
  pull_request:
    paths:
      - 'docs/**'
      - 'docfx.json'

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Setup DocFX
        run: |
          wget https://github.com/dotnet/docfx/releases/download/v2.59.4/docfx.zip
          unzip docfx.zip -d docfx
          
      - name: Build Documentation
        run: |
          ./docfx/docfx build --warningsAsErrors
          
      - name: Check for Warnings
        run: |
          warning_count=$(./docfx/docfx build 2>&1 | grep -c "warning")
          if [ $warning_count -gt 0 ]; then
            echo "❌ Found $warning_count warnings"
            exit 1
          else
            echo "✅ No warnings found"
          fi
