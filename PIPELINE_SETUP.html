Azure DevOps Pipeline Setup

This document explains how to set up the two pipelines for the XMPro documentation.

Prerequisites

  1. Azure DevOps project with access to create pipelines
  2. GitHub PAT (Personal Access Token) with repo permissions
  3. Branch policies configured on main branch

Pipeline Configuration

1. PR Validation Pipeline

This pipeline validates documentation builds on PRs.

File: azure-pipelines-pr.yml

Setup Steps:

  1. In Azure DevOps, go to Pipelines → New Pipeline
  2. Select your repository
  3. Choose "Existing Azure Pipelines YAML file"
  4. Select /public-documentation/azure-pipelines-pr.yml
  5. Name it: "XMPro Documentation - PR Validation"
  6. Save (don't run yet)

Branch Policy Configuration:

  1. Go to Repos → Branches
  2. Click "..." on main branch → Branch policies
  3. Under "Build Validation":
    • Add build policy
    • Select "XMPro Documentation - PR Validation"
    • Set to Required
    • Policy requirement: "Build must succeed"

2. Deployment Pipeline

This pipeline deploys documentation after PR merge.

File: azure-pipelines.yml

Setup Steps:

  1. In Azure DevOps, go to Pipelines → New Pipeline
  2. Select your repository
  3. Choose "Existing Azure Pipelines YAML file"
  4. Select /public-documentation/azure-pipelines.yml
  5. Name it: "XMPro Documentation - Deploy"
  6. Add required variables:
    • Go to Variables → New variable
    • Name: DOCS_DEPLOY_PAT_CLASSIC
    • Value: Your GitHub PAT token
    • Keep as secret: ✓
  7. Save

How It Works

  1. Developer creates PR → PR Validation pipeline runs automatically
  2. PR build passes → Branch policy satisfied, PR can be merged
  3. PR is merged → Deployment pipeline triggers on main branch
  4. Manual approval → Approvers decide which versions to deploy
  5. Deployment → Approved versions are deployed to GitHub Pages

Testing

Test PR Pipeline

  1. Create a test PR
  2. Verify PR pipeline runs and completes
  3. Check branch policy shows as satisfied

Test Deployment Pipeline

  1. Merge a PR to main
  2. Verify deployment pipeline triggers
  3. Test approval process
  4. Verify deployment to GitHub Pages

Troubleshooting

PR Pipeline Issues

  • Not triggering: Check PR trigger in yaml
  • Branch policy not satisfied: Ensure pipeline name matches policy

Deployment Pipeline Issues

  • PAT token errors: Verify DOCS_DEPLOY_PAT_CLASSIC variable
  • Not triggering on merge: Check trigger configuration
  • Approval timeout: Default is 24 hours, can be adjusted

Pipeline Names

  • PR Validation: XMPro Documentation - PR Validation
  • Deployment: XMPro Documentation - Deploy
