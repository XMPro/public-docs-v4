GitBook to DocFX Migration Contributor Guide

Introduction

This guide outlines the process for contributing to the XMPro documentation migration from GitBook to DocFX. Each contributor should follow this standardized workflow to ensure consistency and quality across the migrated documentation.

Setup

  1. Clone the repository and ensure you're on the correct branch (20290-migrate-public-docs)
  2. Familiarize yourself with the MIGRATION-FUNCTION.md document which contains detailed instructions on how to convert GitBook syntax to DocFX
  3. Start the DocFX site locally to preview changes: 
    cd /path/to/public-documentation
docfx docfx.json --serve
    This will build the site and start a local web server at http://localhost:8080

Migration Workflow

1. Claim a Section

Before starting work:

  1. Update the MIGRATION-TRACKER.md file to claim a section
  2. Add your name to the section you're working on
  3. Commit this change to avoid duplicate work by other contributors

2. Page Migration Process

For each page in your assigned section:

  1. Compare the Source and Destination Files

    • Source: /gitbook/docs/path/to/file.md
    • Destination: /docs/path/to/file.md
    • Use MigrateContent(sourcePath, destinationPath) command if available

  2. Identify and Fix Differences

    • Check for GitBook-specific syntax that needs conversion
    • Pay special attention to:
      • Video embeds
      • Images
      • Notes/warnings
      • PDF file references
      • Reference links

  3. Manage Images

    • Verify that all images in the source file exist in the destination's images/ folder
    • Follow the image naming convention (kebab-case)
    • Compare file sizes to ensure they match

  4. Fix Video Embeds

    • Replace GitBook video embeds with proper HTML iframe embeds
    • Example: 
      <div class="video-container">
    <iframe width="560" height="315" src="https://www.youtube.com/embed/VIDEO_ID" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></iframe>
</div>

  5. Handle PDF References

    • Copy PDFs to /docs/assets/pdfs/
    • Update references using the format: 
      [PDF Name.pdf](/docs/assets/pdfs/pdf-viewer.html?pdf=PDF_Name.pdf)

  6. Convert Notes and Warnings

    • Replace GitBook hints with DocFX note syntax: 
      > [!NOTE]
> Important information here

  7. ⚠️ CRITICAL: Rebuild the Site

    • ALWAYS run docfx docfx.json in the public-documentation folder
    • This is REQUIRED after ANY content migration to ensure proper processing
    • If you're not already running the site with --serve, you can rebuild and start the server with: 
      cd /path/to/public-documentation
docfx docfx.json --serve

  8. Verify in Browser

    • Check the rendered page at http://localhost:8080 to ensure everything displays correctly
    • Pay special attention to videos, images, and formatting
    • Test navigation links and any interactive elements

3. Commit Changes

For each page or related set of pages:

  1. Create a single commit with clear, descriptive message
  2. Include details about what was changed
  3. Example commit message format: 
    Migration: Add XMPro Notebook page

- Fixed YouTube embed format
- Verified all images are present and correct
- Updated note syntax to DocFX format

4. Update Tracker

After completing a section:

  1. Update the MIGRATION-TRACKER.md file
  2. Mark your section as complete
  3. Commit this change

Common Issues and Solutions

Video Embeds

  • Problem: DocFX doesn't support GitBook's video embed syntax
  • Solution: Use HTML iframe in a container div (see example in step 4 above)

Images

  • Problem: Missing or incorrectly named images
  • Solution: Copy from GitBook assets folder and rename to kebab-case

Notes/Warnings

  • Problem: GitBook hint syntax doesn't work in DocFX
  • Solution: Convert to DocFX note syntax (see step 6 above)

PDF References

  • Problem: GitBook file references don't work in DocFX
  • Solution: Use the DocFX PDF viewer link format (see step 5 above)

Helpful Tips

  1. Use the MIGRATION-FUNCTION.md as your reference
  2. Always run docfx docfx.json after making changes
  3. Use docfx docfx.json --serve to continuously preview changes as you work
  4. Make focused commits for each page or logical group of pages
  5. When in doubt, compare with already migrated pages for examples
  6. If you encounter errors in the local server, check the console output for details

Contact

If you have questions or encounter issues not covered in this guide, contact the project lead for assistance.
