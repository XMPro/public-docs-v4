Critical Requirements for Release Notes Migration

1. Relative Path Considerations

Release notes files are at depth /docs/release-notes/ so image references should use:

../images/image-name.png

2. Version-Specific Considerations

  • Preserve version number formatting exactly as-is
  • Maintain chronological order of release notes
  • Ensure cross-references between versions work correctly

3. GitBook Syntax Patterns Expected

  • {% hint style="info" %} → > [!NOTE]
  • {% hint style="warning" %} → > [!WARNING]
  • GitBook image embeds → Standard markdown images
  • Video embeds → HTML iframe embeds
