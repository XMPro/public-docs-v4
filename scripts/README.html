Documentation Scripts

This directory contains scripts to help with documentation management and migration.

copy-missing-images.ps1

This PowerShell script copies missing images with snake case names.

Description

During the migration from GitBook to DocFX, image files with spaces in their names may not render correctly. This script checks for missing images referenced in markdown files and creates copies of existing images with the snake case names.

Usage

# Basic usage - copies missing images with snake case names
.\copy-missing-images.ps1

Output

The script provides detailed information about the copying process, including:

  • The number of image files processed
  • The number of image files copied
  • The number of errors encountered
  • Details about each file copied or skipped

Example console output:

Copied: image (595).png -> image_595.png
Skipping: image_107.png (already exists)
Error: Original file not found - Chart example.gif

=== Summary ===
Total files to process: 35
Files copied: 1
Errors: 1
Done!

rename-images-to-snake-case.ps1

This PowerShell script renames image files to lower snake case and updates markdown files with the new image names.

Description

During the migration from GitBook to DocFX, image files with spaces in their names may not render correctly. This script automates the process of renaming these image files to lower snake case (replacing spaces with underscores and converting to lowercase) and updating all markdown files that reference these images to use the new filenames.

Usage

# Basic usage - renames image files and updates markdown files using default parameters
.\rename-images-to-snake-case.ps1

# Use custom image and docs directories
.\rename-images-to-snake-case.ps1 -ImagesDir "C:\path\to\images" -DocsDir "C:\path\to\docs"

# Dry run - shows what would be done without actually renaming files
.\rename-images-to-snake-case.ps1 -DryRun

Parameters

  • ImagesDir: Path to the directory containing the images to rename. Default is '../docs/images'.
  • DocsDir: Path to the directory containing the markdown files to update. Default is '../docs'.
  • LogFile: Path to the log file where the script will write its output. Default is 'rename-images-to-snake-case.log' in the current directory.
  • DryRun: If specified, the script will only show what would be done without actually renaming files.

Output

The script creates a log file with detailed information about the renaming process, including:

  • The number of image files found
  • The number of image files renamed
  • The number of markdown files found
  • The number of markdown files updated
  • Details about each file renamed and updated

replace-placeholder-content.ps1

This PowerShell script replaces placeholder content in DocFX pages with original content from GitBook, converting GitBook-specific markup to DocFX-compatible markdown.

Description

During the migration from GitBook to DocFX, placeholder pages were created for content that needs to be migrated. This script automates the process of replacing these placeholder pages with the original content from GitBook. It also handles the conversion of GitBook-specific markup to DocFX-compatible markdown, including:

  1. Converting image paths from GitBook format (../../.gitbook/assets/image.png) to DocFX format (../images/image.png)
  2. Converting GitBook hint blocks ({% hint style="info" %}...{% endhint %}) to DocFX note blocks (> [!NOTE]...)
  3. Converting GitBook warning blocks ({% hint style="warning" %}...{% endhint %}) to DocFX warning blocks (> [!WARNING]...)
  4. Converting GitBook embed blocks ({% embed url="..." %}) to simple markdown links
  5. Converting GitBook figure blocks with captions to standard markdown images with alt text

The script reads a CSV file containing a list of placeholder pages (generated by find-placeholder-pages.ps1), finds the corresponding GitBook source files, and copies the content from GitBook to replace the placeholder content in DocFX, applying the necessary conversions.

Usage

# Basic usage - replaces placeholder content using default parameters
.\replace-placeholder-content.ps1

# Use a custom CSV file
.\replace-placeholder-content.ps1 -CsvPath "custom-placeholder-pages.csv"

# Specify custom GitBook and DocFX paths
.\replace-placeholder-content.ps1 -GitBookRoot "C:\path\to\gitbook\docs" -DocFxRoot "C:\path\to\docfx\docs"

Parameters

  • CsvPath: Path to the CSV file containing the list of placeholder pages. Default is 'placeholder-pages.csv' in the parent directory.
  • GitBookRoot: Path to the root directory of the GitBook documentation. Default is '../gitbook/docs'.
  • DocFxRoot: Path to the root directory of the DocFX documentation. Default is '../docs'.
  • LogFile: Path to the log file where the script will write its output. Default is 'replace-placeholder-content.log' in the current directory.

Output

The script creates a log file with detailed information about the replacement process, including:

  • The number of placeholder pages processed
  • The number of successful replacements
  • The number of failed replacements
  • Details about each page processed, including the DocFX path, GitBook path, and whether the replacement was successful

Example console output:

find-placeholder-pages.ps1

This PowerShell script identifies placeholder markdown files in the documentation and exports the list to a CSV file.

Description

During the migration from GitBook to DocFX, placeholder pages were created for content that needs to be migrated. This script helps identify these placeholder pages by searching for the text:

This is a placeholder page for content that needs to be migrated.

*This file was automatically generated by the check-missing-pages script.*

The script generates a CSV file with the following information:

  • FileName: The name of the markdown file
  • FilePath: The relative path to the file from the public-documentation directory
  • Directory: The directory containing the file

It also provides a summary showing the distribution of placeholder files by directory.

Usage

# Basic usage - finds placeholder pages and exports to CSV
.\find-placeholder-pages.ps1

# Open the CSV file automatically after generation
.\find-placeholder-pages.ps1 -OpenCsv

Output

The script creates a CSV file named placeholder-pages-[timestamp].csv in the public-documentation directory.
