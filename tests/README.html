DocFX Documentation Testing Framework

This directory contains comprehensive testing approaches for the XMPro documentation site that has been migrated from GitBook to DocFX.

Testing Categories

1. DocFX Build Validation

  • Build Error Detection: Ensures DocFX builds without any errors
  • Warning Analysis: Identifies and fails on any DocFX warnings including:
    • InvalidFileLink: Broken internal links, missing images, and invalid file references
    • InvalidBookmark: Broken anchor/bookmark references within documents
  • Metadata Validation: Checks for missing or invalid frontmatter metadata
  • Cross-reference Validation: Verifies all internal document references resolve correctly

2. Visual Regression Testing

  • Screenshot Comparison: Detects visual changes in rendered pages
  • Responsive Design Testing: Validates documentation on different screen sizes
  • Theme Consistency: Ensures consistent styling across pages

Test Scripts

  • validate-docfx-build.ps1 - DocFX build validation with error/warning detection (includes link and image validation)
  • visual-regression.ps1 - Visual regression testing setup
  • run-all-tests.ps1 - Runs all validation tests in sequence

Running Tests

Prerequisites

  • PowerShell 7.0+
  • Node.js 16+ (for visual regression tools)
  • DocFX installed and configured

Quick Start

# Run all validation tests
./run-all-tests.ps1

# Run specific test category
./validate-docfx-build.ps1  # Includes link and image validation
./visual-regression.ps1     # Visual testing only

CI/CD Integration

DocFX build validation with strict error and warning checks is integrated into the main Azure Pipeline (azure-pipelines.yml). The build will fail if any warnings or errors are detected, preventing deployment of documentation with issues.
