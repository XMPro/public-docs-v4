DocFX Documentation Testing Framework

This directory contains comprehensive testing approaches for the XMPro documentation site that has been migrated from GitBook to DocFX.

Testing Categories

1. Link and Reference Validation

  • Broken Link Detection: Validates all internal and external hyperlinks
  • Image Reference Validation: Ensures all image references point to existing files
  • Cross-reference Validation: Checks DocFX cross-references between pages

2. Content Integrity Testing

  • GitBook to DocFX Comparison: Compares content structure between source and target
  • Markdown Syntax Validation: Ensures proper markdown formatting
  • Metadata Validation: Checks required frontmatter and page metadata

3. Visual Regression Testing

  • Screenshot Comparison: Detects visual changes in rendered pages
  • Responsive Design Testing: Validates documentation on different screen sizes
  • Theme Consistency: Ensures consistent styling across pages

4. Search Functionality Testing

  • Search Index Validation: Ensures all content is properly indexed
  • Search Result Accuracy: Tests search relevance and results
  • Search Performance: Monitors search response times

Test Scripts

  • validate-links.ps1 - Comprehensive link and image validation
  • compare-content.ps1 - GitBook to DocFX content comparison
  • visual-regression.ps1 - Visual regression testing setup
  • validate-images.ps1 - Image reference and file validation
  • search-test.ps1 - Search functionality testing

Running Tests

Prerequisites

  • PowerShell 7.0+
  • Node.js 16+ (for visual regression tools)
  • DocFX installed and configured

Quick Start

# Run all validation tests
./run-all-tests.ps1

# Run specific test category
./validate-links.ps1
./compare-content.ps1 -SourcePath "../gitbook" -TargetPath "../docs"

CI/CD Integration

Tests are designed to run in Azure Pipelines. See azure-pipelines-tests.yml for the test pipeline configuration.
